## **문제 1 : [원소들의 곱과 합](https://school.programmers.co.kr/learn/courses/30/lessons/181929)**

### 문제내용
> 정수가 담긴 리스트 `num_list`가 주어질 때, 모든 원소들의 곱이 모든 원소들의 합의 제곱보다 작으면 1을 크면 0을 return하도록 solution 함수를 완성해주세요.

### 문제풀이
```python
def solution(num_list):
    temp1 = 1  # 모든 원소들의 곱
    temp2 = 0  # 모든 원소들의 합의 제곱

    for i in num_list:
        temp1 *= i
        temp2 += i
    temp2 = temp2 ** 2

    return 1 if temp1 < temp2 else 0
```
> 모든 원소를 곱하고 더하는 행위 자체는 리스트 원소를 모두 훑는 것이므로 loop를 이용해 모든 원소를 곱하고 더한 값을 별도 객체로 저장해주고 비교해준다.

## **문제 2 : [2차원으로 만들기](https://school.programmers.co.kr/learn/courses/30/lessons/120842)**

### 문제내용
>정수 배열 `num_list`와 정수 `n`이 매개변수로 주어집니다. `num_list`를 다음 설명과 같이 2차원 배열로 바꿔 return하도록 solution 함수를 완성해주세요. <br> `num_list`가 [1, 2, 3, 4, 5, 6, 7, 8] 로 길이가 8이고 `n`이 2이므로 `num_list`를 2 * 4 배열로 다음과 같이 변경합니다. 2차원으로 바꿀 때에는 `num_list`의 원소들을 앞에서부터 n개씩 나눠 2차원 배열로 변경합니다.

|num_list|n|result|
|:-----------------:|:-----:|:---------------------:|
|[1, 2, 3, 4, 5, 6, 7, 8]|2|[[1,2], [3,4], [5,6], [7,8]]|

### 문제풀이
```python
def solution(num_list, n):
    length = len(num_list)

    answer = [[num_list[i*n + j] for j in range(n)] for i in range(int(length / n))]
    return answer
```

### 나의 옛날 풀이
```python
def solution(num_list, n):
    answer = [[0] * n for i in range(int(len(num_list)/n))]

    for i in range(int(len(num_list) / n)):
        for j in range(n):
            answer[i][j] = num_list[i*n+j]

    return answer
```
>1. 옛날에는 문제에서 요구하는 2차원 리스트에 shape로 0을채워넣어 만들어서 인덱싱을 하나씩 읽어가면서 문제에서 요구하는 `num_list`에 원소로 채워넣어가는 방식이였다. 해당 방식은 2차원 리스트에서 answer 인덱싱을 앞에서부터 호출하면 어떤 위치에 원소가 호출되는지에 대한 이해를 잘 해야 오류가 발생하지 않는 방식
>2. 비슷하지만 listcomprehension에 익숙해져서 일반화가 잘된 방식 코드 가독성도 매우 좋아짐

## **문제 3 : [문자열밀기](https://school.programmers.co.kr/learn/courses/30/lessons/120921)**

### 문제내용
> 문자열 "hello"에서 각 문자를 오른쪽으로 한 칸씩 밀고 마지막 문자는 맨 앞으로 이동시키면 "ohell"이 됩니다. 이것을 문자열을 민다고 정의한다면 문자열 `A`와 `B`가 매개변수로 주어질 때, `A`를 밀어서 `B`가 될 수 있다면 밀어야 하는 최소 횟수를 return하고 밀어서 `B`가 될 수 없으면 -1을 return 하도록 solution 함수를 완성해보세요.


### 문제풀이 
```python
def solution(A, B):
    length = len(A)
    A = A*2
    if B not in A :
        return -1
    for i in range(length):
        if B == A[length -i:2*length - i] :
            return i
```

### 나의 옛날 풀이
```python
def solution(A, B):
    pre = A
    for i in range(len(A)):
        if pre == B :
            return i
        pre = pre[-1] + pre[:len(A) - 1]
    return -1
```
### 다른 사람들의 풀이
```python
solution=lambda a,b:(b*2).find(a)
```


>1. 옛날에는 문자열의 길이만큼 for문을 돌려서 몇번 문자를 밀어야 되는지를 return 하는 방식으로 진행
>2. 최근에 배운거지만 확실한 오류는 미리 먼저 제외를 해주는게 좋음
>3. 문자열을 두배로 늘려 그 안에 B가 없다면 loop를 돌 필요 없이 바로 -1을 return 하는 방식으로 과거 방식은 굳이 loop를 다 돌아야 -1을 return해주지만 현재 풀이는 처음에 -1인지 판단을 먼저하기 때문에 비용적인 측면에서 더욱 유리하다.
>4. 다른 사람들의 풀이를 보면 나와는 다른 a*2를 하는 것이 아닌 b*2를 하여 a를 찾는다면 몇번 문자를 밀어야 되는지 find를 통해 찾을 수 있고 없으면 -1을 반환해주는 find에 특성상 엄청나게 간결하게 작성가능
